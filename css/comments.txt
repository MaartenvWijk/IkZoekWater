Goede morgen,

Ik heb je script bekeken. Ziet er goed uit! De uitwerking voldoet aan de opdracht, dus netjes gedaan.

Ik heb een aantal opmerkingen nav het reviewen van je code. Hopelijk helpen deze je weer een stapje verder!

[1] Jouw function checkKeyPress heeft meerdere taken/verantwoordelijkheden

Op basis van de functienaam zou ik verwachten dat je alléén checkt of keypress heeft plaatsgevonden (in dit geval bijvoorbeeld of de "enter toets" is ingedrukt.
Naast het feit dat je dát in deze functie checkt, doe je ook "meer": je controleert of het getal juist is; je reset velden als dat nodig is
Hoewel dit niet "per se" helemaal fout is, is het goed om hier geregeld eens goed naar te kijken. Voor herbruikbaarheid van code zou je beter deze functie kunnen opsplitsen in meerdere functies. Bijvoorbeeld deze functie; 1 die de getal-controle doet en 1 die de velden reset. Je roept in je huidige functie dan de nieuw gemaakte functies aan.
Als je dan de controle van juistheid van getallen wilt koppelen aan *zowel* de druk-op-de-knop áls aan "het verlaten van een veld" (dat je een rood tekstje achter de invoer toont als het fout is) dan kun je dmv deze zelfde functie checken. Dan weet je ook zeker dat de checks hetzelfde zijn (consistentie)

[2] Kleinigheidje: controle op "13" mag 13 zijn

Je schrijft   if (key.keyCode == "13") . Echter numerieke waarden horen niet tussen quotes ("). Dus je kunt deze beter weglaten. Het script werkt er overigens wel om: javascript is blijkbaar forgiving ;--)

[3] use strict

Heel slim dat je "use strict" gebruikt
Ik zie dat je dat in elke functie doet.
Als je dat zó doet werkt het, maar moet je het "per functie" doen: het heeft namelijk "locale scope" in die functie.

Wat handiger is, is om "use strict" helemaal boven in in je javascript te definieren (buiten de functies (erboven dus)). Daarmee heeft het geen local maar global scope. Dat betekent dat het dan geldt voor het hele script INCLUSIEF de functies ( en hoef je het daarin niet nogmaals te definieren ).

Zie deze link voor meer info: https://www.w3schools.com/js/js_strict.asp

[4] console.log

Positief punt: ik vind het heel goed dat je gebruik hebt gemaakt van console.log. Dat is een hele goede manier om je javascript code te "debuggen".
Als je je ontwikkelwerk "klaar" hebt, kun je het beter weghalen (het is overhead). Maar voor ontwikkeldoeleinden is het heel handig. (en: prima om ze er in inzendingen gewoon in te laten staan hoor! maar ik dacht: goed om even te noemen). Anders geef je ook meer info weg aan eindgebruikers dan nodig (security aspect)



Roberto
